# 대기 정책 (Waiting Policy)

> 결정된 항목은 ✅, 미결 항목은 ❓로 표시합니다.

---

## 1. 용어 정의

| 용어 | 의미 |
|---|---|
| 대기 등록 | 기구 사용을 위해 줄을 서는 행위 |
| 사용 요청 | 현재 기구 사용자에게 "빨리 끝내달라"는 독촉 알림 전송 |
| 내 차례 알림 | 앞 사람이 사용 완료해 내 순서가 됐을 때 받는 알림 |

---

## 2. 전체 플로우

```
기구 선택
  └─ GoalSetting (세트 수 / 휴식 시간 설정)
       └─ WaitRequest (대기 인원 확인) → "대기 등록하기"
            └─ [WAITING 등록 API] → 대기열 추가
                 └─ Waiting (대기 중 화면)
                      ├─ "사용 요청 보내기" 가능 (독촉 알림)
                      ├─ "대기 취소" → SelectEquipment
                      └─ 내 차례 됐을 때
                           └─ 토스트 "예약한 {기구명}에 자리가 비었어요!" + 이동 버튼
                                └─ [5분 내 미응답 → 다음 대기자로 넘김]
                                └─ 이동 버튼 탭 → WaitRequest 화면 (운동 시작 확인)
                                     └─ "운동 시작하기" → [WAITING → USING 전환]
                                          └─ 세트 타이머 화면
                                               ├─ 마지막 세트 완료 → 완료 화면 (자동)
                                               └─ 중도 취소 → 완료 화면
                                                    └─ "확인" → SelectEquipment
                                                         └─ 다음 대기자에게 내 차례 알림
```

---

## 3. 결정 필요 항목

### 3-1. 버튼 이름 / 역할 구분

- **WaitRequest 페이지 버튼**: ✅ `대기 등록하기`
- **Waiting 페이지 버튼**: `사용 요청 보내기` — 현재 기구 사용자에게 독촉 알림 전송

---

### 3-2. 사용 요청 (독촉 알림) 정책

✅ 쿨다운 5분, 최대 3회 유지  
✅ 현재 기구를 사용 중인 사람이 없을 경우 (대기 1번 = 내가 첫 번째) → 버튼 비활성 아니라 토스트로 "지금 바로 사용할 수 있어요! 내 차례입니다." 안내

---

### 3-3. 내 차례 알림

✅ Socket.io 앱 내 알림만 (브라우저 Push 알림 없음)  
✅ 내 차례가 되면 토스트로 "예약한 {기구명}에 자리가 비었어요!" + 이동 버튼 표시 (자동 화면 전환 없음)  
✅ 사용자가 토스트의 이동 버튼을 탭하면 WaitRequest 화면으로 이동  
✅ 5분 내 응답 없으면 다음 대기자에게 순서 넘어감 → BE에서 타이머로 처리

---

### 3-4. 기구 사용 완료 처리

✅ 완료 방식: 세트 타이머 화면에서 마지막 세트가 끝나면 자동으로 완료 화면 이동  
✅ 강제 완료: 사용 시작 후 1시간 초과 시 BE에서 자동으로 COMPLETED 처리 → 다음 대기자에게 순서 넘김  

**완료 화면 표시 내용** (구버전 동일)
- 박수 이미지 + "{기구명}을/를 멋지게 성공하셨군요!"
- 운동 시작 날짜
- 총 운동시간 / 총 휴식시간
- "확인" 버튼 → SelectEquipment로 이동

---

### 3-5. 동시 대기 제한

✅ 현재 정책 유지 — 1인 1기구, 동시 대기 불가

---

### 3-6. 대기 취소

✅ 모든 상태에서 취소 가능

| 상태 | 취소 가능 | 이동 화면 |
|---|---|---|
| 대기 중 (WAITING) | ✅ | SelectEquipment |
| 내 차례 알림 후 | ✅ | SelectEquipment |
| 사용 중 (USING) | ✅ | 완료 화면 |

---

### 3-7. 대기 예상 시간 계산

✅ `대기 인원 수 × 10분` 고정 유지  
(세트당 실제 운동 시간은 사람/기구마다 달라 데이터 기반 계산이 의미없음)

---

## 4. 참고 — 현재 상태 (DB 스키마)

```
WaitingQueue
  - status: WAITING | USING | COMPLETED | CANCELLED
  - queuePosition: 줄 순서
  - sets / restSeconds: GoalSetting에서 입력한 값
```

`USING` 상태가 스키마에 있으나 현재 FE/BE 모두 미구현.
